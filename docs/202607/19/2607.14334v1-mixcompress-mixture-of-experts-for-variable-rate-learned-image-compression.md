## 摘要
学习图像压缩（LIC）的瓶颈在于需要为每个率失真操作点存储独立模型。现有的可变比特率（VBR）方法旨在通过密集参数调制来减少这一开销，但强制共享骨干网络近似发散映射会导致严重的特征纠缠。具体而言，低速率平滑梯度与高频纹理细节的保留天然存在冲突，导致性能次优。为解决此问题，我们提出MixCompress，一种基于稀疏结构特化的统一VBR框架。尽管稀疏门控的混合专家（MoE）路由成功缓解了梯度冲突，但其在固定计算预算下运行。为应对高比特率增加的表示需求，我们引入混合深度（MoD）扩展以动态缩放模型容量。结合条件辅助变换（CAT）进行动态子带能量调制，我们的分层框架有效实现了容量动态缩放。大量实验表明，MixCompress不仅能够匹配单独优化的单速率基线，甚至可以超越它们，为计算高效的图像编码建立了新的帕累托前沿。

## Abstract
Learned image compression (LIC) is bottlenecked by the need to store independent models for each rate-distortion operating point. Existing variable bit-rate (VBR) methods aim to reduce this overhead via dense parameter modulation, but forcing a shared backbone to approximate divergent mappings causes severe feature entanglement. Specifically, low-rate smoothing gradients inherently conflict with the preservation of high-frequency textural details, leading to sub-optimal performance. To resolve this, we propose MixCompress, a unified VBR framework based on sparse structural specialization. While sparsely gated Mixture-of-Experts (MoE) routing successfully mitigates gradient conflict, it operates on a fixed computational budget. To address the increased representational demands of higher bit-rates we introduce a Mixture-of-Depths (MoD) extension to dynamically scale model capacity. Combined with Conditional Auxiliary Transforms (CAT) for dynamic sub-band energy modulation, our hierarchical framework effectively dynamically scales capacity. Extensive evaluations demonstrate that MixCompress not only matches individually optimized single-rate baselines but can even surpass them, establishing a new Pareto frontier for computationally efficient image coding.

# 中文详细总结：MixCompress

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：现有学习图像压缩（LIC）方法需为每个率失真操作点独立训练与存储一个模型，导致训练成本、存储开销和部署复杂度剧增。已有可变比特率（VBR）方法通过密集参数调制（如条件卷积、潜在缩放）共享一个骨干网络，但强制共享参数迫使模型在低速率平滑与高速率细节保留之间做出妥协，引发严重的**梯度冲突**（gradient conflict）和**特征纠缠**（feature entanglement），性能显著低于对应单速率独立模型。
- **动机**：克服密集参数调制的局限性，通过**稀疏结构特化**实现统一、高效的可变比特率模型，使其性能匹配甚至超越单速率基线。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：引入**稀疏门控混合专家（MoE）** 和**混合深度（MoD）** 模块，结合**条件辅助变换（CAT）**，实现基于目标比特率 λ 的动态容量分配与梯度隔离。
- **关键技术细节**：
  - **MoE块**：在分析变换、合成变换和超先验网络的末尾瓶颈层替换为 MoE 模块。每个块包含一个全局共享专家（保证稳定信息流）和 N 个路由专家（本论文 N=4，K=2）。路由器仅依赖速率嵌入 λe（与输入图像无关），输出 top-K 专家索引。训练时通过**噪声退火**策略（向路由器 logits 添加可衰减的高斯噪声）防止专家坍塌。
  - **MoD扩展**：为支持高速率更高的表示需求，专家按深度递增（例如嵌套 Bi-RWKV 块）。路由器根据 λe 选择不同复杂度的专家，动态扩展容量。
  - **条件辅助变换（CAT）**：在编码器和解码器中引入基于 λe 的 Haar 小波变换子带能量调制。CAT 保留 AuxT 的快捷结构，但通过 FiLM 网络生成尺度/偏置参数，动态调整能量压缩程度。解码端采用逆缩放（iCAT）保持对称。
  - **梯度冲突缓解机制**：通过稀疏路由，不同率点的梯度主要在所选专家的交集上相互作用。由于高速率与低速率选中的专家高度不重叠，冲突梯度被隔离。实验验证了层间梯度余弦相似度分布从负长尾向零附近偏移。
  - **推理优化**：由于路由和 CAT 仅依赖 λe，可预先计算并缓存专家指派和调制参数，推理时无需额外计算。

- **数据集**：OpenImages 前 400K 图像（训练），Kodak（24 张，768×512）、CLIC Professional 验证集（41 张）与测试集（60 张，高至 2K）、Tecnick（100 张，1200×1200）进行评测。
- **基准/锚点**：VTM-23.1（传统编码器），BD-Rate（PSNR）为指标。
- **对比方法**：
  - **单速率基线**：Hyper-prior、Minnen18、Cheng20、ELIC、TCM、FAT、LALIC（均独立训练 6 个率点）。
  - **可变速率基线**：LALIC-QRAF（量化误差感知）、LALIC-CondConv（条件卷积），均在同一骨干 LALIC 上复现。
  - **MixCompress变体**：MixCompress-LALIC-MoE、MixCompress-TCM-MoE、MixCompress-LALIC-MoD、MixCompress-TCM-MoD。

## 6. 主要结论与发现

- MixCompress 作为统一可变速框架，**在所有四个数据集上匹配甚至超越**单独优化的单速率 LALIC 基线（MoD 变体超越所有单速率对比方法）。
- 稀疏结构路由成功**缓解梯度冲突**，使不同率点的优化轨迹解耦。
- QRAF 等密集调制方法在 Bi-RWKV 骨干上因 LayerNorm 导致率敏感信号丢失，性能严重退化；MixCompress 通过结构分离避免此问题。
- **MoD 优于 MoE**，深度自适应容量扩展带来额外收益。
- 推理开销增加极小（编码+0.023s，解码+0.006s），支持实际部署。

## 7. 优点

1. **方法创新**：首次将稀疏 MoE 与 MoD 系统性地引入可变速率 LIC，创造性解决梯度冲突这一核心障碍。
2. **性能卓越**：不仅消除可变速率性能差距，还**超越单速率最优基线**，建立新帕累托前沿。
3. **训练高效**：一次训练覆盖所有率点，总时长仅相当于训练一个单率模型，大幅降低训练和存储成本。
4. **推理高效**：路由和 CAT 参数可缓存，推理零开销，仅需少量额外 FLOPs（+10~40%）即可获得多率支持。
5. **消融充分**：逐步揭示各组件（CAT、MoE、MoD）的贡献，验证设计合理性。
6. **可视化分析**：通过梯度相似度、通道利用率等深入解释原理。

## 8. 不足与局限

1. **实验覆盖范围有限**：仅测试了 MSE 损失下的 PSNR 指标，未涉及 MS-SSIM（虽补充提及，但未给出主要表格）；未与近期感知优化编解码器（如 JPEG AI）做直接对比（作者也说明不直接可比）。
2. **骨干依赖性**：方法主要验证在 LALIC 和 LIC-TCM 两个骨干上，未来需更多架构验证（如 ELIC、FAT）。
3. **专家数量与稀疏度选择**：固定 N=4、K=2，未深入探讨不同配置对性能-效率权衡的影响（仅在补充材料提及）。
4. **实际部署考虑**：虽然推理开销低，但需加载全部专家参数到显存（MoD 总计 156.55M），对显存受限场景可能仍存在压力。
5. **应用限制**：当前框架主要面向基于变换编码的 LIC，对于其他压缩范式（如生成式压缩）的迁移性未涉及。
6. **缺乏理论证明**：梯度冲突缓解的直观解释和统计验证充分，但缺乏严格的理论保证或收敛性分析。
