## 摘要
混合专家模型（MoE）是一类流行的大语言模型（LLM），具有高效率和准确性。然而，在KV缓存密集型的服务场景中，MoE模型往往在模型权重的GPU内存需求与不断增长的KV缓存之间存在紧张关系。我们提出PagedWeight，一种新颖的MoE LLM服务管理方法，它在运行时动态量化MoE模型的权重，并平衡专家权重精度与KV缓存大小。PagedWeight揭示并有效导航了模型任务准确性、内存消耗和吞吐量/延迟之间的复杂权衡。在多个内存敏感的MoE服务场景中，PagedWeight在质量-内存权衡上优于几个现有的量化基线。PagedWeight实现了FP16等效精度，同时节省高达72.0%的GPU内存并提升1.94倍的吞吐量，在相似内存预算下，其质量比量化方法提升高达39.3%，且吞吐量损失最多4.1%。

## Abstract
Mixture-of-Experts (MoE) is a popular class of large language models (LLMs), offering high efficiency and accuracy. However, in KV-cache-intensive serving scenarios, MoEs often exhibit a tension between the GPU memory requirements of the model weights and the growing KV cache. We propose PagedWeight, a novel management method for MoE LLM serving that dynamically quantizes MoE model's weights at runtime and balances expert-weight precision with the KV cache sizes. PagedWeight exposes and effectively navigates the complex tradeoff between the model's task accuracy, memory consumption, and throughput/latency. Across several memory-sensitive MoE serving scenarios, PagedWeight improves the quality-memory tradeoff over several existing quantization baselines. PagedWeight achieves FP16-equivalent accuracy with up to 72.0% GPU memory savings and 1.94$\times$ throughput improvement, and improves quality over quantization methods by up to 39.3% at a similar memory budget with at most 4.1% throughput loss.

## 2. 论文提出的方法论

### 2.1 核心思想
- 将每个专家线性块（linear-block）的任意精度（Any-Precision）位平面和查找表（LUT）视为“权重页面”（weight page），并维护一个权重页面表（Weight Page Table），记录每个线性块的当前位宽（committed bitwidth）和目标位宽（desired bitwidth）。
- 运行时根据KV缓存压力，由质量感知规划器（Quality-Aware Runtime Planner）决定哪些专家的精度需要降低，并将冗余位页面异步卸载到CPU RAM；当内存压力缓解时再重新加载。
- 提出融合混合精度MoE内核，支持每个线性块不同位宽的同时执行，避免精度切换带来的性能开销。

### 2.2 关键技术细节
1. **权重页面表示**：
   - 每个线性块 `i = (ℓ, e, u)`（层、专家、线性块类型 gate_up/down）存储一组支持位宽 `B_i`、当前位宽 `q_i`、目标位宽 `d_i`、每个位宽的GPU内存大小 `M_i^b`。
   - 释放内存量 `Δ_i^{b→b'} = M_i^b - M_i^{b'}` (b' < b)。

2. **质量感知规划器**：
   - **离线全局敏感度**：基于海森加权的量化误差作为敏感度分数 `s_i^b`，定义降精度损伤 `g_i^{b→b'} = max(s_i^{b'} - s_i^b, 0)`。
   - **在线路由统计**：收集专家路由质量（routing mass），将专家按路由质量分桶，热专家的损伤乘数更大、位宽下限更高。
   - **提示残差**：训练线性回归头，根据每个序列的输入特征（路由加权均值和最大范数）预测当前提示对量化损伤的修正因子 `η_i^{b→b'}`。
   - **规划目标**：将KV缓存压力转换为需要释放的字节数 `D`，对每个候选动作计算每字节损伤 `bd/Δ`，贪心选择代价最低的动作直到满足 `D`。
   - **算法流程**（算法1）：在每个规划步，更新计划队列；当压力超过阈值时，从最新计划中选择并执行页面移动。

3. **异步页面移动**：
   - 规划器持续生成计划，当压力触发时，异步将不需要的位平面从GPU拷贝到CPU，同时降低提交位宽；恢复时先拷贝回GPU再提高位宽。
   - 使用融合MoE AP内核，直接读取不同位宽页面执行，避免精度转换延迟。
