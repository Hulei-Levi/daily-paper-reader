## 摘要
我们研究了利用交互感知的混合专家模型，通过结构化健康记录的多层次视图进行中风后僵硬预测。尽管性能提升甚微，路由归因揭示了不同视图间系统性的重要性差异，强调了视图构建是解释性的关键。

## Abstract
We study interaction-aware mixture-of-experts for post-stroke rigidity prediction using multi-level views of structured health records. Despite minimal performance gains, routing attribution reveals systematic importance differences across views, underscoring view construction as key to interpretability.

- **研究动机**：结构化健康数据（如计费代码、诊断、手术、生命体征等）是临床预测的常用基体，但现有模型大多将其视为单一表格输入，隐含地学习交互，缺乏明确的交互建模与可解释性。论文探索是否可以通过将同一份结构化记录转化为多种替代表示（多视图），并显式建模视图间的交互，来提升预测性能并增强解释性。
- **整体含义**：尽管多视图建模带来的性能提升有限，但路由归因揭示了不同视图构建方式下系统性的重要性差异，表明视图构建本身是可解释性的关键杠杆，即使预测增益微弱。

## 2. 论文提出的方法论

- **核心思想**：采用**交互感知混合专家模型（I2MoE）**，该模型基于部分信息分解（PID）框架，将来自多个视图的信息分解为独特（unique）、冗余（redundant）和协同（synergistic）信号，并通过路由网络动态分配样本到不同专家。
- **关键技术细节**：
  - 对于同一份结构化健康记录，定义三种层次的多视图：
    - **模型级视图**：将完整记录分别输入三个表格神经网络（SAINT, NODE, DCN-V2），取其隐藏表示作为视图。
    - **数据级视图**：按临床语义将特征分为管理、程序、诊断、体征、ICD指示共5个组，每组经MLP编码。
    - **表示级视图**：分别用GATv2（图）、NODE（表格）、Qwen3-Embedding（结构化文本渲染）得到三个嵌入。
  - I2MoE 结构：每个视图对应一个独特专家（unique expert），加上一个协同专家（synergy expert）和一个冗余专家（redundancy expert）。重加权网络 `g` 根据所有视图的拼接输出各专家的权重：
    ```
    ŷ = σ( Σ_m w_u^m f_u^m(h_m) + w_s f_s(h) + w_r f_r(h) )
    ```
    其中 `[w_u^1,...,w_u^M, w_s, w_r] = g([h_1;...;h_M])`。
  - 损失函数：任务损失（交叉熵） + 弱监督交互损失 `λ_int L_int`，鼓励专家捕获特定类型的信号。
- **算法流程**：
  1. 对每个患者，按三种视图定义分别提取视图表示 `{h_m}`。
  2. 拼接视图表示，输入重加权网络获得各专家权重。
  3. 各独特专家根据对应视图输出预测，协同专家处理所有视图，冗余专家处理所有视图的共享部分。
  4. 加权求和后经sigmoid得到最终预测，并计算总损失。

## 6. 论文的主要结论与发现

1. **预测性能提升有限**：多视图I2MoE在部分指标上显著优于XGBoost，但增益很小（如模型级AUROC从0.7627升至0.7714）。数据级视图甚至未显著优于基线。
2. **交互专家并非性能关键**：去除协同和冗余专家后，性能无显著下降，说明独特专家和重加权网络已能捕获大部分预测信号。
3. **视图构建系统性影响路由分配**：
   - 模型级视图大量激活协同和冗余专家（因为不同骨干网络表示重叠度高）。
   - 数据级视图也依赖协同和冗余（临床分组间不独立）。
   - 表示级视图更多分配到独特专家（图、表格、文本嵌入保留更多视图特有信息）。
4. **局部路由一致性**：在表示级模型中，相似患者的专家权重分配比随机患者更接近，为基于路由的个体化解释提供基础。

## 7. 优点

- **方法创新**：将多视图学习与部分信息分解（PID）结合，通过混合专家形式显式建模独特、冗余、协同信号，为结构化健康数据的可解释性提供了新视角。
- **路由归因**：不同于传统注意力或SHAP，路由权重直接指示样本对每种交互类型的依赖，且具有全局和局部一致性。
- **实验设计周密**：30次随机种子复现，统计显著性检验；消融实验全面；多视图层次覆盖了模型、数据和表示三个层面。
- **临床相关性**：针对实用性强的真实临床任务（卒中后僵硬预测），数据规模大（~13万次住院），且使用公开数据库，结果可复现性强。

## 8. 不足与局限

- **性能增益微弱**：多视图方法在主要指标上提升极小或没有，且交互专家贡献可以被独特专家“替代”，降低了方法的应用价值。
- **任务与数据集单一**：仅针对一个二分类任务、一个数据集，泛化性存疑。需要更多任务（如死亡率、再入院）和更多数据库验证。
- **视图构建不够独立**：所有视图源自同一记录，本质是同一信息的不同编码，导致独特性有限，可能解释了为什么协同和冗余专家贡献不大。
- **缺乏与其他融合方法对比**：未与简单拼接、加权融合、多模态变压器等基线比较，无法定位I2MoE在融合策略中的相对优势。
- **可解释性深度不足**：路由权重虽提供视图级归因，但未连接到更细粒度的临床概念（如具体诊断或指标），对临床用户的直接帮助可能有限。
- **计算与资源需求**：模型级视图需训练三个独立表格网络，表示级需三个不同编码器（GATv2、NODE、嵌入模型），训练开销大但回报小。
- **风险与限制**：论文明确说明方法仅用于研究，不能替代临床判断，需经过外部验证和监督。
