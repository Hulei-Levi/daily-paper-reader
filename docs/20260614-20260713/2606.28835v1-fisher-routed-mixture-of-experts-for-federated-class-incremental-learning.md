## 摘要
联邦学习（FL）作为一种有前途的分布式机器学习范式而出现。然而，将FL扩展到类增量学习场景带来了独特的挑战：1）共享模型过载导致的容量冲突和灾难性遗忘，2）非独立同分布（Non-IID）数据带来的异质性，以及3）同步的类别错位。在本文中，我们提出了用于联邦类增量学习的费舍尔路由专家混合模型（FedFMX），这是一个通过跨客户端的自适应专家专业化来应对这些挑战的新框架。关键见解是将每个样本路由到一个专家子集，该子集共同优化知识获取和保留。具体来说，我们引入了一个费舍尔路由专家评分（FRES）模块，通过基于费舍尔的稳定性成本和基于梯度的可塑性增益来估计专家重要性。然后，我们设计了一个自适应专家选择（AES）模块，通过量化边际贡献来自适应地确定专家子集。最后，通过路由感知正则化（RAR），我们实现了负载均衡和高效的FL训练。我们从理论上证明了$\mathcal{O}(T^{-1})$的收敛速度。在多个基准上与最先进方法的广泛实验比较证明了FedFMX的优越性。

## Abstract
Federated Learning (FL) emerged as a promising distributed machine learning paradigm. However, extending FL to the class incremental learning scenarios introduces unique challenges: 1) Capacity conflict and catastrophic forgetting from the shared model overloading, 2) Heterogeneity from Non-Independent and Identically Distributed (Non-IID) data, and 3) Synchronized class misalignment. In this paper, we propose \textbf{F}isher-Routed \textbf{M}i\textbf{X}ture of Experts for \textbf{Fed}erated Class-Incremental Learning (\textsc{FedFMX}), a novel framework to address these challenges via adaptive expert specialization across clients. The crucial insight is to route each sample to an expert subset that jointly optimizes knowledge acquisition and retention. Specifically, we introduce a Fisher-Routed Expert Scoring (FRES) module to estimate expert importance via Fisher-based stability cost and gradient-based plasticity gain. Then, we design an Adaptive Expert Selection (AES) module by quantifying marginal contributions for adaptive expert subset determination. Finally, by the routing-aware regularization (RAR), we achieve load balance and efficient FL training. We theoretically prove the $\mathcal{O}(T^{-1})$ convergence rate. Extensive experiments on multiple benchmarks compared with state-of-the-art methods demonstrate the superiority of \textsc{FedFMX}.

### 1. 核心问题与整体含义（研究动机和背景）
- **问题**：联邦类增量学习（FCIL）需要在保护数据隐私的前提下，让多个客户端协作、增量式地学习新类别。这带来三大挑战：  
  - **容量冲突与灾难性遗忘**：共享全局模型被增量数据重载，旧知识被覆盖。  
  - **统计异质性**：客户端数据非独立同分布（Non-IID），导致梯度方向不一致。  
  - **同步类错位**：不同客户端在不同阶段遇到新类别，造成标签空间临时不一致，加剧模型漂移。  
- **动机**：现有FCIL方法通常将全局模型视为整体，缺乏细粒度的稳定性-可塑性调节，且无法适应客户端动态演化的任务轮廓。  
- **整体含义**：本文旨在通过专家模块化和自适应路由，在保留旧知识的同时高效吸收新知识，实现稳定、协作的FCIL。

### 2. 方法论：核心思想、关键技术细节、算法流程
- **核心思想**：将模型分解为共享骨干 + 多个专家子空间，对每个样本动态激活一个专家子集，联合优化知识获取与保留。  
- **关键技术细节**：  
  - **FRES（Fisher-Routed Expert Scoring）模块**：  
    - 为每个专家维护对角Fisher信息矩阵（指数移动平均平方梯度）。  
    - 定义**稳定性成本** \(\Delta_c^{(k)} = \frac{1}{2} \| \text{diag}(\tilde{\mathbf{F}}_k)^{1/2} \tilde{\mathbf{g}}_k \|_2^2\)，惩罚重要历史方向的当前梯度。  
    - 定义**可塑性增益** \(\Delta_g^{(k)} = \|\mathbf{g}_k\|_2^2 / (\text{Tr}(\tilde{\mathbf{F}}_k) + \epsilon)\)，鼓励在平坦参数区域见效快的专家。  
    - 在激活前计算这两个指标，避免选择过拟合或脆弱的专家。  
  - **AES（Adaptive Expert Selection）模块**：  
    - 将专家选择视为合作博弈，定义价值函数 \(v(S) = -\ell_S - \alpha \sum_{k\in S}\Delta_c^{(k)} + (1-\alpha)\Delta_g(S)\)，融合准确性、遗忘风险和可塑性。  
    - 计算每个专家的**Shapley边际贡献** \(\phi_k\)，按降序排序后迭代添加专家，直到边际增益低于阈值 \(\mu\)（动态确定子集大小）。  
  - **RAR（Routing-Aware Regularization）**：  
    - **负载均衡正则化**：最小化软路由分布熵 \(L_{\text{bal}}\)，防止少数专家被过度使用。  
    - **蒸馏式门控**：将Shapley值转化为soft教师分布 \(q(k|x)\)，轻量级门控网络 \(g_\eta\) 输出学生分布 \(p(k|x)\)，通过KL散度 \(L_r\) 蒸馏，推理时仅用门控网络无需再计算Shapley值。  
  - **总损失**：\(\mathcal{L} = \mathcal{L}_{\text{task}} + \beta L_{\text{bal}} + \gamma L_r\)，其中激活集内专家按正边际贡献加权。  
- **算法流程**（文字说明）：  
  1. 客户端本地训练时，对每批样本：  
     a. 通过FRES计算每个候选专家的 \(\Delta_c\) 和 \(\Delta_g\)。  
     b. AES依据Shapley值动态选择最优子集 \(\mathcal{K}_b\)。  
     c. 在 \(\mathcal{K}_b\) 上前向/反向传播，更新专家和骨干参数。  
     d. 同时优化门控网络以匹配教师分布。  
  2. 服务端聚合所有客户端的专家和骨干参数（经典FedAvg）。  
  3. 推理时仅用门控网络选择Top-K专家，无需重复计算FRES/AES。  
- **理论分析**：证明在标准FL假设下，FedFMX达到 \(\mathcal{O}(T^{-1})\) 收敛速率（定理1）。

- FedFMX在所有设置下（QLI/DLI、细/粗粒度）持续超越所有baseline，尤其在高异质性和复杂数据集上优势明显。
