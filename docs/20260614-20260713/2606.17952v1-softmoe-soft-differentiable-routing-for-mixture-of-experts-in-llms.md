## 摘要
稀疏混合专家（MoE）架构通过在top-$k$路由下仅激活一小部分专家，在固定推理预算下扩展大语言模型参数。虽然这保持了因果性并适用于自回归语言模型，但离散的top-$k$算子不可微分，迫使每个输入固定数量的激活专家，导致计算利用率低下。我们提出SoftMoE，用截断的软top-$k$ LapSum松弛替换离散路由，允许基于梯度的专家路由优化。我们进一步参数化每层平均激活专家数量并施加全局预算约束，使模型能够学习如何跨层分配专家容量。SoftMoE与自回归建模完全兼容，在语言建模和下游任务上达到与稀疏MoE相当或更好的性能，同时显著减少激活的专家数量。值得注意的是，学习到的分配高度非均匀，较后的层激活更多专家。源代码已公开$^\dagger$。

## Abstract
Sparse Mixture-of-Experts (MoE) architectures enable scaling LLM parameters under a fixed inference budget by activating only a small subset of experts via top-$k$ routing. While this preserves causality and suits autoregressive language models, the discrete top-$k$ operator is not differentiable, forcing a fixed number of active experts per input and resulting in inefficient use of computation. We propose SoftMoE, which replaces discrete routing with a truncated soft top-$k$ LapSum relaxation, allowing gradient-based optimization of expert routing. We further parameterize the mean number of active experts per layer and impose a global budget constraint, enabling the model to learn how to allocate expert capacity across layers. SoftMoE remains fully compatible with autoregressive modeling and achieves performance comparable to or better than sparse MoE on language modeling and downstream tasks, while activating significantly fewer experts. Notably, the learned allocation is highly non-uniform, with later layers activating more experts. The source code is publicly available$^\dagger$.

# 论文总结：SoftMoE: Soft Differentiable Routing for Mixture-of-Experts in LLMs

## 1. 核心问题与整体含义
- **研究动机**：大规模语言模型的扩展通常依赖增加参数量，但推理成本线性增长。稀疏MoE通过仅激活部分专家（top-k路由）在固定推理预算下扩展参数，但离散top-k算子不可微分，导致每个输入必须激活固定数量的专家，无法根据输入自适应调整；同时专家容量在各层间是均匀预设的，造成计算资源浪费。
- **整体目标**：设计一种可微分、保持因果性且计算高效的MoE路由机制，允许模型自适应地分配专家容量（跨层、跨token），在保持或提升性能的同时减少激活的专家数。

## 2. 方法论
- **核心思想**：用**可微分的软top-k松弛**（基于LapSum函数）替换硬top-k路由，实现梯度可通过专家选择过程传播；进一步**参数化每层平均激活专家数**并施加全局预算约束，使模型自动学习如何分配专家容量。
- **关键技术细节**：
  - **SoftTopK算子**：利用LapSum函数（Laplace累积分布函数之和）求解一个连续阈值$\tilde{x}$，使得$\sum_i F_{\text{Lap}}(r_i - \tilde{x}) = k$（$r_i$为路由分数），输出软选择权重$\tilde{p}_i = F_{\text{Lap}}(r_i - \tilde{x}) \in [0,1]$，且$\sum \tilde{p}_i = k$。该计算复杂度$O(n)$，全可微。
  - **截断（Truncation）**：为保持稀疏性，对低于阈值$\tau$的权重修剪（类似ReLU），使激活专家数量由输入动态决定。
  - **学习每层分配**：引入可学习参数$\eta_l$，通过softmax+仿射变换映射到每层平均激活数$k_l = \pi_l \cdot (K - L) + 1$，其中$\sum k_l = K$（全局预算），$k_l \ge 1$。该参数化端到端可微，模型可在训练中重新分配各层容量。
- **整体流程**：每个token经门控网络得到路由分数，SoftTopK输出软权重，截断后激活部分专家，输出为加权专家输出和。所有组件（包括$k_l$）均可通过梯度下降优化。

## 3. 实验设计
- **数据集**：
  - 预训练：OpenWebText (OWT, ~9B tokens) 和 C4 (~206B tokens)。均使用BPE tokenizer（词表32k）。
  - 下游零样本评测：PIQA（物理常识）、HellaSwag（句子补全）、ARC-E（科学问答）。
- **基准（Baselines）**：
  - 标准稀疏MoE（Switch Transformer风格），固定每层激活专家数$k \in \{1,2,3,4\}$。
  - SoftMoE变体：不带学习分配的SoftMoE*（固定$k$），带学习分配的SoftMoE（初始$k \in \{1.5, 2\}$，全局预算$K$对应平均2或3等）。
- **模型架构**：10层decoder-only GPT-2，每层32个专家（每个专家5M参数），总参数量1.63B。所有模型共享相同基础结构和训练协议（优化器、学习率、热身、余弦退火等）。

## 6. 主要结论与发现
- **效率优势**：SoftMoE（不含学习分配）在相同或更低的语言建模损失下，激活专家数平均减少17%–24%（训练和推理均观测到）。
- **自适应专家激活**：软路由允许模型根据输入动态激活不同数量的专家，简单token激活少，复杂token激活多，且训练中激活数自适应变化。
- **学习分配模式**：在全局预算约束下，模型自发地将更多专家分配给**最后几层**（后3层吸收约50%预算），早期和中间层专家较少。该模式在训练早期迅速形成（<25k步）。
- **性能匹配或超越**：在PIQA、HellaSwag上，SoftMoE普遍优于或持平稀疏MoE；在ARC-E上部分配置更好；语言模型损失一致更低。
- **计算开销**：LapSum复杂度$O(n)$，相对专家计算可忽略；实际训练吞吐量接近稀疏MoE。

## 7. 优点
- **创新性**：首次将LapSum软top-k松弛应用于MoE路由，实现完全可微的专家选择，并引入可训练的每层容量分配——这是标准MoE无法做到的。
- **自回归兼容**：每个token独立处理，无token混合，完全兼容因果语言模型，避免了Soft MoE（Puigcerver et al.）中的跨位置混合问题。
- **简洁高效**：LapSum具有封闭形式解，无迭代，计算和内存线性于专家数；截断机制可动态控制稀疏度。
- **实验严谨**：对比了多个预算水平，包含消融研究，并公开代码。

## 8. 不足与局限
- **规模限制**：实验仅在1.63B参数模型上进行（10层，32专家），未验证更大规模（如数十亿参数）时的行为。
- **语言覆盖**：仅使用英语语料（OWT和C4）及英语下游任务，未探索多语言或多模态场景。
- **评估覆盖**：下游任务有限（仅3个），且ARC-E结果方差较大；未评测代码生成、数学推理等能力。
- **潜在偏差**：学习分配模式是否对特定任务/模型结构敏感？未实验不同层数或不同专家数。
- **效率分析**：实际推理时软路由会导致激活专家数波动，对批处理和硬件优化可能引入新挑战（虽然文中提到上界控制）。
