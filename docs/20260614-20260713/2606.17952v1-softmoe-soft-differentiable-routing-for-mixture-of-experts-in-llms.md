## 摘要
稀疏专家混合（MoE）架构通过在推理预算固定时仅通过top-k路由激活一小部分专家，从而使得大语言模型参数得以扩展。虽然这保持了因果性并适用于自回归语言模型，但离散的top-k算子不可微，导致每个输入需激活固定数量的专家，且计算效率低下。我们提出SoftMoE，该方法用截断的软top-k LapSum松弛替代离散路由，允许基于梯度的专家路由优化。我们进一步参数化每层平均激活专家数，并施加全局预算约束，使模型能够学习如何跨层分配专家容量。SoftMoE与自回归建模完全兼容，在语言建模和下游任务上达到与稀疏MoE相当或更优的性能，同时显著减少激活的专家数量。值得注意的是，学习到的分配高度非均匀，较后层激活更多专家。源代码已公开提供^\dagger。

## Abstract
Sparse Mixture-of-Experts (MoE) architectures enable scaling LLM parameters under a fixed inference budget by activating only a small subset of experts via top-$k$ routing. While this preserves causality and suits autoregressive language models, the discrete top-$k$ operator is not differentiable, forcing a fixed number of active experts per input and resulting in inefficient use of computation. We propose SoftMoE, which replaces discrete routing with a truncated soft top-$k$ LapSum relaxation, allowing gradient-based optimization of expert routing. We further parameterize the mean number of active experts per layer and impose a global budget constraint, enabling the model to learn how to allocate expert capacity across layers. SoftMoE remains fully compatible with autoregressive modeling and achieves performance comparable to or better than sparse MoE on language modeling and downstream tasks, while activating significantly fewer experts. Notably, the learned allocation is highly non-uniform, with later layers activating more experts. The source code is publicly available$^\dagger$.

# 论文总结：SoftMoE: Soft Differentiable Routing for Mixture-of-Experts in LLMs

## 1. 核心问题与整体含义（研究动机和背景）
- **研究动机**：稀疏 MoE 架构通过 top‑k 路由只激活少量专家，在固定推理预算下扩展模型参数。但硬 top‑k 算子**不可微**，导致每个输入必须激活固定数量的专家，计算效率低下，且无法根据层或 token 自适应分配专家容量。
- **背景**：现有替代方案（如专家选择 token、线性分配、token 混合）往往破坏自回归因果性、或引入推理‑训练不一致、或导致负载不均。因此需要一种**可微、因果兼容、计算高效**的路由机制，允许梯度优化专家选择并自适应分配容量。

## 2. 方法：SoftMoE
### 核心思想
用截断的软 top‑k **LapSum 松弛**替代离散 top‑k 路由，实现端到端可微的专家选择；并参数化每层平均激活专家数，施加全局预算约束，让模型**学习跨层容量分配**。

### 关键技术细节
- **SoftTopK 算子**：基于 LapSum 函数（累积 Laplace 分布之和），通过解方程 `LapSum(x) = k` 得到可微阈值，再取 `F_{Lap}(r_i - x)` 作为软选择权重。输出值在 [0,1] 且和为 k，可微且线性复杂度 O(n)。
- **截断机制**：对低于阈值 τ 的权重设为零，恢复稀疏性，同时允许输入依赖的专家激活数（移除固定 k 限制）。
- **可学习容量分配**：对每层引入参数 η_l，通过 softmax 重参数化得到非负、和为 K 的每层预算 k_l，约束 `Σ k_l = K` 且 `k_l ≥ 1`。全局预算 K 控制总计算量，梯度可流过 k_l 实现端到端优化。
- **自回归兼容性**：每个 token 独立处理，不跨 token/输入混合，保持因果性。

## 6. 主要结论与发现
- **效率提升**：SoftMoE（不含学习分配）在多数配置下达到与稀疏 MoE 相当或更低的 perplexity，同时激活专家数减少 17%‑24%。
- **可学习分配模式**：当允许学习层间预算时，模型将更多容量分配给**后几层**（前 3 层消耗约 50% 总预算），后期层激活更多专家，符合语义层次深化的直觉。
- **性能匹配/超越**：在 HellaSwag 上 SoftMoE 始终优于稀疏 MoE；PIQA 上相当或更好；ARC-E 上在 OWT 上最佳，C4 上相当。
- **训练稳定性**：软路由未引入额外不稳定，激活数在训练中自适应演变。

## 7. 优点
- **可微路由**：首次将基于 LapSum 的软 top‑k 成功应用于大规模自回归 MoE，实现端到端梯度优化。
- **自适应容量分配**：通过可学习、全局约束的专家预算，允许模型按层分配计算，揭示了非均匀分配模式。
- **保持因果性**：不跨 token/输入混合，适合自回归生成。
- **计算开销小**：LapSum 复杂度 O(n)，与标准门控矩阵乘法相比可忽略。
- **兼容现有框架**：基于 Megatron‑LM 实现，易于集成。

## 8. 不足与局限
- **规模限制**：仅在 1.63B 参数模型上实验，未验证更大规模（如 100B+）的泛化性。
- **数据与任务单一**：只涉及英语文本（OWT, C4）和三个常识推理基准，缺乏多语言、知识问答、数学推理等任务。
- **截断机制的非可微边界**：截断阈值 τ 处梯度为零或未定义，可能影响部分专家选择的优化。
- **负载均衡**：仅使用辅助损失，未系统分析极端 token 下的负载极端情况。
- **计算量报告不完全**：缺少显存占用、对比稀疏 MoE 中软路由带来的相对计算开销（除 O(n) 外）。
- **下游实验方差**：ARC-E 结果波动较大，可能与评测集较小有关。
